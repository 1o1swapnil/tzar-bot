# config/env-allowlist.txt — operator-maintained allow-list for env-reader.py
#
# env-reader.py is the ONLY approved way to read secrets (for both the MCP
# read_env tool and Bash callers). Only variable names matching this list (or a
# key declared in .env.example) may be read; anything else is refused as DENIED
# and its value is never touched. This blocks prompt-injection-driven reads of
# arbitrary environment variables (e.g. AWS_SECRET_ACCESS_KEY, SSH keys).
#
# These are MERGED ON TOP of the keys declared in .env.example — this file only
# adds; it never removes. Missing file = .env.example keys only.
#
# Rules:
#   - one variable name OR fnmatch glob per line (case-sensitive)
#   - globs: USER_*_TOKEN matches USER_A_TOKEN, USER_B_TOKEN, ...
#   - lines starting with '#' and blank lines are ignored
#   - override per-invocation with $TZAR_ENV_ALLOWLIST (comma-separated)
#   - point elsewhere with $TZAR_ENV_ALLOWLIST_FILE
#
# Keep entries tight — a broad glob like *_TOKEN defeats the purpose.

USER_*_TOKEN
